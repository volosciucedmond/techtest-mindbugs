Hotel Review Intelligence System
===================================

Short description
-----------

Engineered a backend service using a **local LLM** and **vector embeddings** to automate guest feedback analysis. Developed a semantic deduplication logic that converts repetitive reviews into unique, actionable pain points, providing managers with a structured view of operational performance.

Tech Stack
--------------

*   **Backend:** FastAPI (Python)
    
*   **LLM Integration:** Ollama (running llama3.2:1b)
    
*   **Embeddings:** sentence-transformers (all-MiniLM-L6-v2)
    
*   **Infrastructure:** Docker & Docker Compose
    

Setup
---------------------

### Prerequisites

*   **Docker** and **Docker Compose** installed on your machine.
    

### 1\. Start the System

Navigate to the root directory of the project in your terminal and run:

Bash

`   docker compose up --build   `

> **Heads-up:** On the initial run, the ollama-pull service will automatically fetch the llama3.2:1b model. This might take a few minutes. It's best to keep an eye on the API container logs. Once they give you the "ready" signal, you're good to go.

### 2\. Test the API

Once the containers are running and the Ollama model is fully loaded, you can trigger the file processing endpoint.

**Using FastAPI Interactive Docs:**

1.  Open your browser and navigate to: [http://localhost:8000/docs](https://www.google.com/search?q=http://localhost:8000/docs)
    
2.  Expand the POST /process\_file endpoint.
    
3.  Click **"Try it out"** and then **"Execute"**.
    

## Example Output
Because I'm using a lightweight 1b parameter model alongside the `all-MiniLM-L6-v2` embedding model at a strict 0.8 threshold, the exact deduplication counts will reflect the mathematical reality of those specific models. 

Here is an example of the actual JSON response generated by this pipeline:

```json

```

Architecture Notes
----------------------

*   **Semantic Deduplication:** I utilised the all-MiniLM-L6-v2 model with a 0.8 cosine similarity threshold for precise deduplication. It successfully merges repetitive "Clean room" instances whilst preserving unique semantic nuances, ensuring the data stays high-signal and meets the project requirements.
    
*   **Extraction Stability:** I enforced strict string types to handle LLM schema errors. This keeps the embedding model stable and ensures the final API response remains reliable and predictable.
